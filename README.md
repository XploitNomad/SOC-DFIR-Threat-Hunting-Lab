# SOC-DFIR-Threat-Hunting-Lab

Windows DFIR & Threat Hunting Lab


Project Overview

This project demonstrates how to build a basic SOC/DFIR home lab using free tools to simulate attacker activity, collect Windows logs, and investigate suspicious behavior. The lab was built on a Windows 11 machine using Splunk, Sysmon, and Atomic Red Team.

The objective was to understand how security analysts monitor endpoints, investigate alerts, hunt for suspicious activity, and collect forensic evidence from Windows systems.

By following the steps below, you can recreate the same lab and perform your own investigations.

# Procedures

Step 1: Install Splunk Enterprise

Download and install Splunk Enterprise on your Windows machine.

After installation:

Start Splunk.
Create an administrator account.
Access the Splunk web interface.

Default URL:

http://localhost:8000

Verify that Splunk is running successfully before moving to the next step.

---

Step 2: Install Sysmon

Download Sysmon from Microsoft Sysinternals.

Extract the files and open Command Prompt as Administrator.

Install Sysmon:

Sysmon64.exe -accepteula -i

Verify the installation:

sysmon64.exe -c

You should now see Sysmon events being generated under:

Applications and Services Logs
 └─ Microsoft
     └─ Windows
         └─ Sysmon
             └─ Operational

Sysmon will provide detailed logs for process creation and command execution that will later be sent to Splunk.


----
Step 3: Install Splunk Universal Forwarder

Install Splunk Universal Forwarder on the same Windows machine.

The Universal Forwarder is responsible for sending Sysmon logs to Splunk.

Create or edit the inputs configuration and add:

[WinEventLog://Microsoft-Windows-Sysmon/Operational]
disabled = 0
index = sysmon
renderXml = true
source = XmlWinEventLog:Sysmon

Restart the Splunk Universal Forwarder service after saving the configuration.

Once configured correctly, Sysmon logs should begin appearing in Splunk.


----

Step 4: Generate Activity

To create data for investigation, simulate common attacker behavior using Atomic Red Team and manual commands.

Run the following commands:

powershell.exe
cmd.exe
nslookup github.com
ping google.com

These commands generate process creation and network-related events.

To simulate persistence, create a scheduled task:

schtasks /create /tn WindowsUpdateCheck /tr notepad.exe /sc onlogon

This creates a task that launches Notepad whenever a user logs in.

-----

Step 5: Investigate Activity in Splunk

After generating activity, open Splunk and begin searching for evidence.

Search for PowerShell Activity
index=sysmon powershell.exe

This search identifies PowerShell execution events.

Search for Command Prompt Activity
index=sysmon cmd.exe

This search identifies Command Prompt execution events.

Search for DNS Activity
index=sysmon nslookup.exe

This search identifies DNS lookup activity generated by nslookup.

Search for Application Execution
index=sysmon calc.exe

This search can be used to identify execution of applications such as Calculator.

----

Step 6: Perform Forensic Analysis
Analyze Prefetch Files

Windows Prefetch files can provide evidence of program execution.

Use PECmd to analyze the Prefetch directory:

PECmd.exe -d "C:\Windows\Prefetch" -q --csv "C:\DFIR-Lab\Timeline"

Review the generated output for executed applications.

Examples observed during testing:

POWERSHELL.EXE
NOTEPAD.EXE

This helps confirm that applications were executed on the system.

----

Review Windows Event Logs

Use Event Viewer to review:

Applications and Services Logs
 └─ Microsoft
     └─ Windows
         └─ Sysmon
             └─ Operational

These logs provide additional details about process execution and system activity.

----

Review PowerShell History

PowerShell command history can provide useful evidence during investigations.

Run:

Get-Content (Get-PSReadLineOption).HistorySavePath

This displays previously executed PowerShell commands.

Conclusion

This lab demonstrates how to build a simple Windows DFIR and Threat Hunting environment using free tools. By combining Sysmon, Splunk Enterprise, Splunk Universal Forwarder, Atomic Red Team, and PECmd, it is possible to generate realistic activity, collect logs, investigate suspicious behavior, and analyze forensic artifacts.
